# Руководство пользователя: Автоматический шаблон описания проблем

> **Область применения:** Регламент обязателен для всех участников проектных команд (аналитиков, разработчиков, QA).\
> **Версия процесса:** 1.0 (совместимо с YouTrack 2021.4+)

## 1. Назначение процесса

Данная автоматизация предназначена для внедрения единого корпоративного стандарта оформления дефектов, ошибок и системных аномалий. 

При создании новой задачи робот YouTrack автоматически вставляет в поле `Описание` структурированный markdown-шаблон. Это гарантирует, что автор задачи сразу предоставит инженерам всю критически важную информацию, что сокращает время на разбор (триаж) тикетов, исключает лишнюю переписку и ускоряет исправление багов.

## 2. Структура вставляемого шаблона

В момент нажатия кнопки **«Создать задачу»** поле описания автоматически заполняется следующими логическими блоками:

* **Вводная часть (Памятка):** Временный текст-напоминание о необходимости проверить бэклог на наличие дубликатов и использовать систему голосования.
* **Шаги воспроизведения:** Нумерованный список (`1.`, `2.`, `3.`) для фиксации точной последовательности действий, приводящих к ошибке.
* **Ожидаемый результат:** Четкое описание того, как система *должна* работать согласно ТЗ или документации.
* **Фактический результат:** Описание того, что происходит *вместо этого* (текст ошибки, зависание интерфейса, некорректные данные).
* **Финальная часть (Вложения):** Напоминание о важности прикрепления логов, скриншотов или видеозаписей экрана.

## 3. Инструкция по заполнению для авторов задач

Для успешной модерации и быстрой передачи задачи в разработку соблюдайте следующие правила:

1. **Сохраняйте разметку:** Не удаляйте жирные заголовки (`**Какие шаги...**` и т.д.). Вносите свой текст строго под ними.
2. **Пишите шаги детально:** Описывайте действия пошагово, начиная с исходного состояния (например: *1. Открыть карточку сотрудника; 2. Очистить поле СНИЛС; 3. Нажать кнопку Сохранить*).
3. **Очищайте служебный текст:** Первый абзац (приветствие) и последний абзац (напоминание про логи) являются подсказками. **Перед финальным сохранением задачи обязательно удалите первый и последний абзацы**.
4. **Прикрепляйте доказательства:** Перетаскивайте скриншоты, скринкасты и файлы журналов (логов) напрямую в тело карточки.
